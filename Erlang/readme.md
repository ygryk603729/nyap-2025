# Клиент-серверное взаимодействие: Файловый сервер (Erlang)

## Постановка задачи
Реализован простой TCP-файловый сервер и клиент на языке Erlang.

- **Сервер** слушает подключения на порту 8080.
- По простому HTTP-подобному GET-запросу (`GET /filename HTTP/1.1`) сервер отправляет содержимое запрошенного файла.
- Если файл существует — возвращает 200 OK с содержимым.
- Если файла нет — возвращает 404 Not Found.
- После отправки ответа соединение закрывается (Connection: close).

- **Клиент** запрашивает файл по имени у сервера и сохраняет полученное содержимое в указанный файл.

Клиент и сервер написаны на чистом Erlang с использованием библиотеки `gen_tcp` из OTP.

## Структура проекта
- `server.erl` — код сервера
- `client.erl` — код клиента
- Тестовые файлы для сервера:
  - `file1.txt` — маленький текстовый файл
  - `file2.txt` — файл среднего размера (сотни строк)
  - `file3.txt` — большой файл (10 МБ, созданный командой `fsutil file createnew file3.txt 10485760`)

## Компиляция
В папке с файлами выполните:
```powershell
erlc server.erl
erlc client.erl
